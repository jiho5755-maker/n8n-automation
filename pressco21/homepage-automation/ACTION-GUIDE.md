## 강사 신청 처리 방법

### 신청 접수 → 처리 전체 흐름

```
고객이 신청 폼 작성
  ↓ (FA-002가 1시간 내 자동 알림)
@Pressco21_makeshop_bot 텔레그램 알림 수신
  ↓
NocoDB(https://nocodb.pressco21.com)에서 증빙서류 확인
  ↓
─────────────────────────────────────────────────────
│ 승인하는 경우           │ 반려하는 경우              │
│                         │                            │
│ [진행 상태] → 승인대기  │ [반려 사유] 칸에 사유 입력 │
│                         │ [진행 상태] → 반려         │
─────────────────────────────────────────────────────
  ↓ (FA-001이 5분 내 자동)    ↓ (FA-003이 5분 내 자동)
메이크샵 강사그룹 변경       고객 이메일로 반려 안내 발송
NocoDB → 승인완료 자동 변경   텔레그램으로 발송 완료 알림
승인 축하 이메일 발송
텔레그램 완료 알림
```

### NocoDB 사용 방법

1. https://nocodb.pressco21.com 접속
2. `강사공간` 테이블 클릭
3. 신청 레코드 클릭 → 우측 상세 패널에서 증빙서류 확인

**진행 상태 옵션:**

| 상태 | 설명 | 누가 변경 |
|------|------|---------|
| 신청완료 | 고객이 폼 제출 시 자동 설정 | 자동 |
| 승인대기 | 관리자가 승인 결정 시 변경 | **운영자가 직접** |
| 승인완료 | FA-001이 등급 변경 후 자동 변경 | 자동 |
| 반려 | 관리자가 부적격 판단 시 변경 | **운영자가 직접** |
| 대기중 | 서류 보완 요청 등 임시 보류 | 운영자가 직접 |

> **반려 시 주의사항**: '반려 사유'를 먼저 입력한 뒤 상태를 변경하세요.
> 사유가 비어있으면 이메일 발송이 보류되고 텔레그램으로 보류 알림이 옵니다.

## 시스템 점검 방법

### 워크플로우 실행 확인

- https://n8n.pressco21.com/home/executions 에서 최근 실행 로그 확인
- 빨간색 실패 항목 클릭 → 어느 노드에서 오류가 났는지 확인

### 자주 있는 오류 패턴

| 증상 | 원인 | 해결 |
|------|------|------|
| 텔레그램 알림 안 옴 | 워크플로우 비활성화 또는 서버 다운 | n8n UI에서 Active 상태 확인, `docker compose restart` |
| 메이크샵 등급 변경 실패 | API 권한 또는 허용 IP 문제 | 메이크샵 관리자 → 오픈API 설정 확인 (허용IP: 158.180.77.201) |
| 반려 이메일 미발송 | 이메일 주소 없거나 형식 오류 | NocoDB에서 이메일 주소 확인 후 `n8n_반려알림` 체크 해제 |
| 승인 이메일 미발송 | 이메일 주소 미입력 | NocoDB에서 이메일 추가 → FA-001은 이메일 없으면 이메일만 스킵 (등급은 변경됨) |
